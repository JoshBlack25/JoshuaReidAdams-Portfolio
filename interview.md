# 🧠 Reflective Practice (STAR Method)

This document reflects on my learning experiences during the mock interview, GitHub portfolio development, and general technical growth.

---

## 🎤 MOCK INTERVIEW REFLECTION

### Situation
I participated in a simulated ICT job interview as part of my academic preparation for Work Integrated Learning (WIL).

### Task
I was required to present myself professionally, answer structured interview questions, and demonstrate confidence in both technical and behavioural responses.

### Action
I prepared for the interview by reviewing common ICT interview questions and structuring my responses using the STAR method. I focused on:

- Clear communication
- Professional body language
- Relating answers to real work and academic experiences
- Staying calm under pressure

I also practised linking my answers to real examples from customer service roles and development projects.

### Result
My responses became more structured and confident. I improved my ability to explain technical concepts and behavioural situations clearly without rushing or losing focus.

---

## 💻 GITHUB & MARKDOWN PORTFOLIO REFLECTION

### Situation
I was tasked with building a digital portfolio using GitHub and Markdown as part of my academic assessment.

### Task
The goal was to create a structured, professional portfolio including my CV, projects, and reflective learning evidence.

### Action
I learned and applied Markdown formatting to structure my content into separate files. I organised my repository into:

- `index.md`
- `cv.md`
- `interview.md`
- Assets folder for media

I also prepared the portfolio for GitHub Pages deployment to make it publicly accessible.

### Result
I gained a clear understanding of how developers structure and publish online portfolios. My documentation skills improved, and I became more confident using version control and structured file systems.

## 🔧 GENERAL LEARNING REFLECTION

### Situation
Throughout multiple ICT projects, I was required to continuously learn new tools, languages, and problem-solving techniques.

### Task
I needed to improve my technical ability while managing deadlines, debugging errors, and completing project requirements.

### Action
I applied an iterative learning approach:

- Build small components
- Test functionality
- Identify errors
- Fix and improve continuously

I also used past experience in both academic and workplace environments to guide my decision-making and improve my workflow.

### Result
I became more confident in problem-solving, more patient with debugging, and more structured in my approach to software development tasks.

---

## 🧭 CLOSING REFLECTION

Skill development is not instant. It is built through repetition, correction, and persistence over time.

Progress in development is not loud—it is steady, disciplined, and continuous.
